Project 1: Weather Data Exploration

Dataset: prasad22 — Weather Data (synthetic).

Problem

Explore city-level weather patterns: (1) daily average temperature by city, (2) monthly total precipitation by city, and (3) temperature–humidity relationship.

Data

CSV with columns: Location, Date_Time, Temperature_C, Humidity_pct, Precipitation_mm, Wind_Speed_kmh. Data are synthetic and designed for practice (not real observations).

Pre-processing

  • Parsed Date_Time to datetime; added daily and monthly groupings.
  • Dropped rows missing critical fields (datetime or location).
  • Removed duplicate (Location, Date_Time) pairs.
  • Clipped invalid values (no negative precipitation; humidity within 0–100).

Visualizations

What I Learned

  • Seasonality is clear across cities, with different peak ranges.
  • Precipitation is uneven month-to-month and city-to-city.
  • Temperature and humidity show an inverse pattern; hexbin reveals dense clusters cleanly.

Impact & Limits

This is practice for time-series/EDA and visualization. Because the data are synthetic, don’t draw real-world climate conclusions. Missing features (pressure, cloud cover) limit deeper analysis.

References

  • Kaggle dataset: prasad22 / Weather Data
  • Libraries: pandas, matplotlib
